Simple Installation

To use rslidy for a presentation, the rslidy.js file and all relevant CSS files have to be included in your HTML file, which should also contain the presentation itself.

Alternatively, the rslidy.min.js and rslidy-combined.min.css files can be used if no custom settings or other modifications are needed.

TypeScript and Grunt

Following command can be used to create the final JavaScript file from the original TypeScript file: ``` tsc rslidy.ts ``` Grunt can be used to minify both JS and CSS files and it also creates a ZIP archive of rslidy. Grunt uses following plugins in rslidy: grunt-contrib-uglify, grunt-contrib-cssmin and grunt-zip.

Responsive Web Design

In the overall, rSlidy is responsive and compatible in any device

However, some small design problems were found when doing compatibility tests

After testing testing on a Samsung Galaxy S6, Iphone 5, Iphone 6, Iphone 6 plus, Ipad, Ipad pro, and PC screen with different resolutions:

  • Works smoothly on medium to wide screen hand devices like the SGS6, Iphones 6, 6 plus, but when pressing button "slides", the content of the slide move to the right and some content is actually hidden because it is displaced to leave space for the slides thumbnails at the left
  • In Ipad or Ipad pro, the same problem with the content being displaced takes place. However, when working in horizontal view, the slide content DOES adapt to the slide succesfully
  • In narrow-screen devices as the Iphone 5, we still have the "Slides" button problem, and also the "Menu" button overlaps due to the lack of space in the screen and goes down, below the rest of the menu bar, keeping itself alone
